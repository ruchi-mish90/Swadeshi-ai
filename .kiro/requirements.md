# Requirements for Trade-Off Architect

## Challenge
As a first-year B.Tech student, choosing tech stacks (e.g., Firebase vs. Supabase) feels like guesswork. Need a data-driven tool to replace vibes with engineering.

## Solution
Build a Python-based decision engine using a Weighted Decision Matrix to rank technologies based on Time-to-Market, Scalability, Budget. Explain trade-offs to keep developers in control.

## Key Features
- Load tech data from JSON.
- Collect user weights.
- Calculate and rank scores.
- Display verdict with trade-off reasoning.
- Include testing and profiling for acceleration.
